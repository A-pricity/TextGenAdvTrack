# Execution Plan

## 目标

完成课程实践中的 `生成文本对抗赛道`，优先提交 `生成文本检测赛道` 的有效结果。

## 建议阶段

### 阶段 1：规则与数据确认

- 固化当前已知规则
- 明确 `M4 / HC3` 禁训边界
- 整理网盘数据内容
- 记录允许使用的外部训练集

输出：

- 数据来源清单
- 合规说明

### 阶段 2：先跑通 baseline

- 优先跑通单模型监督微调 baseline
- 目标不是最优，而是先生成合法提交文件
- 推荐首个 baseline：
  - `xlm-roberta-base`
  - 或 `deberta-v3-base`

输出：

- 首版验证集分数
- 首版 `.xlsx` 提交文件

### 阶段 3：补强泛化

- 增加零样本分支或统计特征
- 做简单融合
- 对逃逸样本做误差分析

输出：

- 融合版本分数
- 错误案例分析

### 阶段 4：准备课程提交物

- GitHub：
  - 代码
  - 数据来源说明
  - 运行方法
  - 实验结果
- PPT：
  - 任务背景
  - 技术路线
  - 新颖性
  - 效果
  - 分工

## 推荐优先级

如果时间有限，按下面顺序做：

1. 生成文本检测
2. 合规训练数据
3. 单模型 baseline
4. 本地评测与提交流程
5. 结果可视化
6. 逃逸检测或融合增强

## 当前风险

### 风险 1：训练数据违规

必须避免任何形式把 `M4` 或 `HC3` 当训练集使用。

### 风险 2：只做零样本

纯零样本路线很可能在 AUC 和跨域稳定性上吃亏。

### 风险 3：只关注分类、不关注分数

官方评分重 AUC，说明分数排序质量比单点分类更关键。

### 风险 4：太晚准备 PPT

课程提交不是只有榜单结果，还看方法新颖性、展示质量和分工说明。
